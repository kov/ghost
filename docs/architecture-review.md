# ghost — architecture & design review

**Scope note on anchors** *(corrected 2026-07-16 after verifying git topology — the finalize agent's original "stale snapshot" claim was wrong):* the review was run against `main` at `c86cd74`, which **already includes** the merge of `fix/damage-scroll-offset-underreport` and the two alt-screen-resize commits `c5a046c`/`f94a07b` (both ancestors of HEAD, dated 2026-07-14). Line numbers are cited against the current tree — spot-checked correct (e.g. finding #1, `protocol.rs:54`). The genuine limitation is one of **coverage, not staleness**: the alt-screen-resize region (`buffer.rs`/`line.rs`) was trusted largely unread, so findings touching that region are lower-confidence. See §6.

---

## 1. Executive summary

The system is structurally healthy at its core: the two-emulator/one-byte-stream contract, the Scope A single-owner `SessionState` split, and the Elm-shaped functional core are all real and load-bearing, and the core damage contract turned out to be *better* enforced than the review initially believed (`damage_audit.rs` is exactly the property test one reader proposed, already landed). The single biggest structural tension is that ghost's hardest invariants — damage-claim completeness above the emulator, exactly-once feed delivery, single-answerer for queries, resync-into-fresh-state — are held by convention at seams two or three files away from the state they protect, and every one of them is about to have its mint-sites multiplied by Scope B, the broker, or predictive echo. Six independent readers converged on that shape, and adversarial verification confirmed eight of the top ten findings, several empirically (red tests written and run). The most urgent single defect is a wire-format break already shipped on dev branches: `ClientMsg::Policy` was inserted mid-enum, silently shifting `Hello`/`Subscribe`/`Observe` ordinals against any still-running level-4 host — fixable in days, not after level-5 binaries proliferate. The recurring foreground-render-stall is a bug *class* with three confirmed-open members (OSC 4 palette, connect-prompt baseline advance, self-heal tail holes) plus two non-render mechanisms producing the identical symptom; the permanent fix is extending the core's existing damage enforcement up through the view/present layers, not an eighth hand-added term. The roadmap mostly fits, but two items are blocked as scoped (`per-host broker`, `ghost play`) and Scope B strains on five fronts, none fatal, all cheaply pre-fixable — the recommended sequence front-loads roughly a dozen S-effort guards that convert discipline into structure before any big move.

## 2. Top findings

Confidence tiers used throughout:
- **[verified]** — adversarially re-verified against source, several with empirical scratch tests.
- **[probed]** — a reader ran a throwaway test that went red; not adversarially re-checked.
- **[read]** — static-read inference by one or more readers; not adversarially verified.

### Dropped after verification

- **`sessioninfo-no-serde-defaults` — REFUTED, dropped.** The claimed empty-fleet failure cannot trigger: every field appended to `SessionInfo` over time (`size`, `connection`, `cwd`) is `Option<T>`, which serde already treats as omittable, and git history shows no probe-passing remote ghost ever predates the full struct. What survives is only a *forward-compat footgun*: a future non-`Option` appended field would be silently swallowed by the bare `continue` at `ghost-ui/src/main.rs:473`. Cheap hardening: warn once per host on parse failure, and pin a "new listing fields must be `Option`" rule in a test.

### Verified findings, ranked by leverage

**1. `ClientMsg::Policy` inserted mid-enum shifts wire ordinals — silent misdecode against live level-4 hosts.** `ghost-vt/src/protocol.rs:54`. **[verified — confirmed, empirically]** The verifier reproduced it with the workspace's exact postcard version: a new client's `Hello` decodes on an old host as `Subscribe` (silently — trailing bytes ignored), new `Subscribe` as `Observe`; `Observe` fails loud-ish (decode error → hung observation). Hosts are long-lived daemons, so the coexistence window is first-class supported; the crate's own append-only discipline (`protocol.rs:152`) was violated, contradicting the Policy commit's own compat reasoning. No test pins discriminants. **Collides with:** Scope B, per-host broker — both multiply long-lived cross-version connections. **Fix (S):** move `Policy` to the enum tail + a regression test pinning every variant's encoded discriminant. **Timing is load-bearing:** land it before level-5 binaries proliferate, or ship as `PROTO_POLICY=6` and write off the level-5 window — moving it later breaks running level-5 hosts the same way.

**2. Remote proto gate exists on the wire but is never enforced → infinite reconnect flicker.** `ghost-vt/src/remote.rs:525`. **[verified — confirmed]** `__probe` emits `proto=N` (self-described in a comment as "a future compatibility gate") but `probe()` checks only the marker substring, and `Client::connect_ssh` hardcodes the *local* `PROTO_LEVEL`. An older `ghost` on the remote PATH is preferred over the staged version-matched copy, the first `ClientMsg::Policy` is undecodable, the old host drops, the reconnect probe finds the session alive, `finish_reattach` re-runs the identical handshake — a persistent ~1s attach/drop loop with zero diagnostics. `proto_compat.rs` covers only the local socket path. **Collides with:** broker, predictive echo (every future PROTO bump reopens the window). **Fix (S):** parse `proto=` from the probe reply, thread it through `RemoteHost`/`connect_ssh`, treat under-level as probe failure so negotiate falls through to the staged copy; count reattach cycles and surface persistent failure.

**3. Client-side `Conn::flush` never retries after `WouldBlock` — autonomous query replies strand.** `ghost-vt/src/transport.rs:211`. **[verified — confirmed, empirically]** The transport's own doc prescribes "poll for writability, then flush"; only the host honors it — no `wants_write` caller exists anywhere in ghost-ui. The verifier proved a stranded DA reply stays stranded across ten read-only pumps. Narrowing: *any* subsequent send un-strands it, so the indefinite hang is specifically the idle-user case — a remote child blocked awaiting a DA/DSR/XTGETTCAP reply while the user waits on it. **Collides with:** predictive echo (input-path latency becomes visible), broker (one shared channel strands many sessions' replies). **Fix (S):** `if conn.wants_write() { conn.flush()? }` at the top of `Session::pump`/`Subscriber::pump` — rides the existing 8ms cadence.

**4. Observed fleet tiles' emulators generate query replies; single-answerer holds only because the shell's lookup happens to miss.** `ghost-ui-core/src/fleet.rs:2019`. **[verified — confirmed, empirically]** The fleet's filter strips only `reaches_the_desktop()` commands; a DSR fed to an observed tile emits `Cmd::SendInput` out of the core, dropped silently at `main.rs` because observed sessions aren't in `w.sessions`. No core guard, no covering test, nothing marks this intended. Any refactor giving observed sessions a write path (broker, Scope B routing) double-answers queries, corrupting the child's input stream. **Fix (S):** filter `SendInput` for non-driven tiles in the core (or a `driving` flag on the feed), plus the one-line test.

**5. OSC 4/104 palette changes bypass every damage channel — a live, open member of the stall class.** `ghost-term/src/terminal.rs:863`. **[verified — confirmed, empirically; worse than claimed in one respect]** `SetPalette` dirties nothing; `Presented` snapshots only OSC 10/11/12; a *pure* OSC-4 feed emits no `Cmd::Redraw` at all, and a mixed feed bands only the written rows while un-banded texture rows keep the old palette. Both diagnostics are blind (verify compares unresolved `Color::Indexed`; the watchdog sees presents landing). Tempering: most palette-setting programs repaint afterwards; pywal-class tools that remap without repainting stay wrong indefinitely. Notably, `set_policy` tightening dirties ALL rows for exactly this recolor-without-cell-writes class — the `SetPalette` omission is an oversight, not a decision. **Explains:** foreground-render-stall (an unclaimed frame input, the exact shape of all ~7 prior fixes). **Fix (S):** claim the viewport on `SetPalette`/`ResetPalette`, or add a palette generation to `Presented`; widen `damage_audit`'s ground truth to (cells, palette).

**6. `mark_presented` advances the damage baseline during the connect prompt — reproducible stale-foreground.** `ghost-ui-core/src/root.rs:2037`. **[verified — confirmed, empirically]** The verifier's scratch test failed exactly as predicted: prompt open (Cmd+G) over a streaming session, feed lands, prompt-scene presents clear `feed_dirty`, Esc → `TermDamage::None` → `Plan::Current` presents the stale pre-prompt texture. Nuance: a *full* freeze needs the stream to quiesce during the prompt; otherwise untouched rows stay stale indefinitely. **Explains:** foreground-render-stall — the strongest single reproducible mechanism found. **Fix (S):** the verifier-preferred form is scene-derived — only mark presented when the presented scene actually contained the foreground Terminal item (also covers `pending_dive` and future non-terminal scenes) — rather than just adding `self.connect` to the guard.

**7. Nothing structural prevents a window from both driving and observing one session — double-feed garbles the emulator unhealably.** `ghost-ui/src/main.rs:2439`. **[verified — confirmed]** Both `Cmd::Observe` arms guard only `w.observers`; no attach path purges observers; the codebase itself documents the hazard at a call site ("driving without adopting double-feeds the tile") while enforcing nothing at the executor. Narrowing: the protection is not solely `FleetModel.mine` — `RootModel.mine` and paired-cmd ordering carry it too — but it's all model-side discipline across a crate boundary. `feed_warm`'s silent-drop arm is the under-delivery mirror. **Collides with:** Scope B (multiplies feed-routing sites). **Fix (S):** `&& !w.sessions.contains_key(&id)` on both arms; debug_assert `sessions ∩ observers = ∅` each pump; breadcrumb on `feed_warm` drops.

**8. Attach resync drops the pending UTF-8 tail — reseeded client renders U+FFFD where the host renders the glyph.** `ghost-vt/src/screen.rs:318`. **[verified — confirmed, empirically]** Reproduced byte-for-byte: braille spinner split across the attach boundary → host `spin: ⠋`, client `spin: ��`, cursors one column apart. Narrowing: only attaches landing inside a split-char window are affected; the cursor heals at the next absolute positioning (a spinner's `\r`), but wrong glyphs persist until overwritten or resynced. Same hole applies to lagged-observer re-seeds. **Collides with:** predictive echo (client==host after reseed is its confirmation precondition). **Fix (S, one line):** append `self.pending` to `resync()` — correct for both GUI emulators (identical buffering) and the transparent-pipe real terminal (simple concatenation). The twin at the recording layer (`checkpoint-pending-utf8-tail`, `server.rs:806`, **[read]**) permanently corrupts checkpoint-based replay and `truncate_before_latest_checkpoint` bakes it in; skip checkpoints while `pending` is non-empty.

**9. Damage completeness above the emulator — NARROWED from the original claim.** `ghost-vt/src/screen.rs:114`. **[verified — partial]** The original "trusted, never enforced" framing is **wrong at the core**: `ghost-term/tests/damage_audit.rs` (landed 2026-07-11) is precisely the proposed property test, cell-diffing random op streams against claimed rows and documenting completeness as the contract; targeted frame-space regressions exist in `ghost-ui-core/terminal.rs` (scroll-offset shift, cap-slide); the watchdog runs always-on with self-heal. What survives: (a) the `Screen::feed` doc comment predates the suite and still disclaims the completeness the tests now pin — a one-paragraph doc fix; (b) `want_redraw` remains a hand-maintained 7-term OR (`ghost-ui-core/src/terminal.rs:1767`), so a *new* pixel-affecting channel can be missed until audited — consistent with the stall's fixed-seven-times history; (c) no *property* harness exists above ghost-term, exactly where the last two under-reports lived. Verify-mode is deliberately log-only by design comment (a diagnostic decision, not neglect) — proposing sampled debug-build enforcement is fine; the original long-term sketch (emulator-side "anything changed" generation) should **not** be adopted as stated: it trades away the banding that per-channel damage exists to enable. **Fix (M):** view-level damage-soundness proptest (ground truth = `layout_frame_at` diff between `mark_presented` calls, covering rows + palette + colors + scroll offsets), plus the doc flip.

## 3. Cross-cutting themes

1. **Damage completeness is enforced at the core, disclaimed in its docs, and unenforced above it.** The corrected picture after verification: `damage_audit.rs` pins the emulator's claim; the view/present layers — where the last two under-reports and both newly confirmed holes (palette, connect-prompt) actually live — have targeted regressions but no property harness, and `want_redraw` remains a hand-grown OR-chain. The stall is a bug class whose members are born wherever a new frame-visible fact skips the enumeration. Enforcement must extend upward *before* the simplification thesis removes the second diff that sometimes masks wrong claims.

2. **Resync assumes a freshly-cleared emulator; three shipped paths feed it warm, and split-state above ghost-term escapes the dump.** Probes went red on first run: duplicated scrollback, alt-screen-stuck mirrors, U+FFFD tails, kitty-stack divergence, orphaned attach-edge queries. The dump-representable invariant is enforced one layer too low — `Screen::pending` and the freshness precondition live above where `prop_dump` can see. Gates predictive echo; makes Scope B's shared state dangerous.

3. **One fact, two owners, synced by hand.** The exact shape Scope A killed survives in five places: fleet `mine` vs `RootModel.mine`, display names (7 paired writes), meta vs descriptor, `ThemeColors::default` vs `Theme::default`, and best-effort policy persistence. Individually cheap; collectively the codebase's dominant defect-manufacturing pattern.

4. **Exactly-once feed and single-answerer are discipline, not structure.** Confirmed empirically on both fronts (#4, #7). Every guard is a convention three files from the state it protects; Scope B multiplies routing sites and the broker would hand observed sessions a write path. All fixes are one-line guards plus debug_asserts.

5. **Wire-format growth is fragile.** Append-only was violated once already (#1), the proto gate is decorative (#2), and the residual `SessionInfo` hazard is a future non-`Option` field silently swallowed by a bare `continue`. Every roadmap item needing protocol growth walks through this; the fixes are S-effort and unblock everything.

6. **The transport is structurally 1:1 per session.** No session envelope, one `io` per `Conn`, a byte-only `__pipe`, and an entire ssh-exec control plane parallel to the framed protocol. ControlMaster is connection reuse, not multiplexing — the broker roadmap item is mis-scoped until this is written down.

7. **Liveness: one slow peer freezes authoritative state or silently eats input.** Blocking PTY write in the host, stranded client flush, unbounded ssh on the event loop, SIGTERM tail drop, reconnect input loss. These are the non-render freezes living inside the stall's reputation.

8. **GPU caches are grow-only with silent failure modes** — the image-substitution incident's own record names the unfinished residue (no eviction, silent re-upload no-op, silent atlas exhaustion, accidental font-key soundness). All become live with font config or image-heavy fleet churn.

9. **The biggest roadmap items lack the test seams to be built test-first — in a codebase where test-first is the law.** No multi-window harness, no real-PTY-under-real-vsync stall reproduction, warm-reseed untested, OSC 52 unpinned, one E2E file leaking into real XDG. Seam-building is on the critical path, not hygiene.

## 5. Recommended refactor sequence

**1. Wire hygiene (all S, independently landable — do this week, before level-5 binaries proliferate).** Move `ClientMsg::Policy` to the enum tail (or re-ship as PROTO 6, writing off the level-5 window) + pin every variant's postcard ordinal in a test; parse `proto=N` from the probe reply and enforce it in `connect_ssh` (under-level PATH ghost falls through to the staged copy); add flush-retry at the top of `Session`/`Subscriber::pump`; warn-per-host on listing parse failure and pin the "new listing fields must be `Option`" rule. **Unlocks:** every protocol append the roadmap needs — predictive echo's PROTO-6, the broker's envelope, bell count — becomes safe against long-lived hosts and un-restaged remotes.
